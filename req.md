# 病院職員管理システム デモアプリ 要件定義書

## 1. プロジェクト概要

### 1.1 目的
ランサーズ案件「病院職員管理システムの構築」への提案用デモアプリ。  
クライアント（徳洲会）にオンライン面談を獲得するため、提案文にデモURLを添付し、他105件の提案との圧倒的な差別化を図る。

### 1.2 技術スタック
- **フレームワーク:** Next.js（App Router）
- **言語:** TypeScript
- **スタイリング:** Tailwind CSS
- **UIコンポーネント:** shadcn/ui
- **チャートライブラリ:** Recharts
- **アイコン:** Lucide React
- **デプロイ先:** Vercel
- **データ:** 全てフロントエンドのモックデータ（API・DB不要）

### 1.3 対応デバイス
- PC（メイン。1280px以上を想定）
- タブレット（補助的にレスポンシブ対応）
- スマートフォン対応は不要

## 2. デザイン方針

### 2.1 カラーパレット

| 用途 | カラー | 備考 |
|------|--------|------|
| 背景（メイン） | `#FFFFFF` | 白ベース、清潔感 |
| 背景（サブ） | `#F8FAFC` | サイドバー、カード背景 |
| プライマリー | `#0F6FDE`（青系） | 医療系の信頼感。ボタン・アクセント |
| プライマリーホバー | `#0B5BBF` | |
| セカンダリー | `#10B981`（緑系） | ステータス「出勤」「承認済」など |
| 警告 | `#F59E0B`（アンバー） | 注意喚起・残業アラート |
| 危険 | `#EF4444`（赤） | エラー・欠勤・期限切れ |
| テキスト（主） | `#1E293B` | |
| テキスト（副） | `#64748B` | |
| ボーダー | `#E2E8F0` | |

### 2.2 デザインルール
- 角丸は `rounded-lg`（8px）を基調とする
- カードには軽い `shadow-sm` を付与し、ホバー時に `shadow-md` へ遷移させる
- セクション間の余白は `space-y-6` を基本とする
- フォントは `Inter`（英数）+ システムフォント（日本語）
- データテーブルは偶数行に `bg-gray-50` のゼブラストライプ
- 全てのボタン・カードにホバーエフェクトを付与（0.2sトランジション）
- ページ遷移はサイドバーのナビゲーションで行い、各ページはスムーズにフェードイン

### 2.3 レイアウト構成
```
┌──────────────────────────────────────────────┐

│ サイド  │  メインコンテンツエリア                 │

│        │                                     │

```

## 3. 共通コンポーネント

### 3.1 サイドバーナビゲーション

背景色 `#F8FAFC`、左端固定。以下のメニュー項目を配置する。

| アイコン | ラベル | 遷移先 |
|---------|--------|--------|
| LayoutDashboard | ダッシュボード | `/` |
| Users | 職員管理 | `/staff` |
| Calendar | シフト管理 | `/shift` |
| Clock | 勤怠管理 | `/attendance` |
| FileText | 休暇申請 | `/leave` |
| Settings | 設定 | （モーダル表示のみ） |

- 現在のページに該当するメニューは、背景色 `bg-blue-50` + テキスト `text-blue-700` + 左端に `border-l-3 border-blue-600` のアクティブ表示
- 各メニューアイコンの右側に、未読件数がある場合は赤いバッジを表示（通知ベル: 3、休暇申請: 2 など）

### 3.2 ヘッダー
- 左端：ロゴ「MediStaff」（フォントウェイト700、青色）
- 右端：通知ベルアイコン（クリックで通知ドロワー表示）、ユーザーアバター+名前「山田 太郎」+ロール「システム管理者」
- ヘッダー下部にはブレッドクラムを表示（例：ホーム > 職員管理 > 職員詳細）

### 3.3 通知ドロワー
ヘッダーの通知ベルをクリックすると、右側からスライドインするドロワーを表示する。

表示する通知例（5件）：
1. 「佐藤花子さんが休暇申請を提出しました」— 10分前
2. 「3月のシフトが未確定です（外科病棟）」— 1時間前
3. 「田中一郎さんの資格（看護師免許）が来月更新期限です」— 3時間前
4. 「先週の残業時間が基準を超過している職員が4名います」— 昨日
5. 「新しい職員（鈴木美咲）の登録が完了しました」— 2日前

### 4.3 シフト管理（`/shift`）

#### 4.3.1 ページ上部：コントロールバー

- **月選択**: 「← 2026年2月 | 2026年3月 | 2026年4月 →」の形式。矢印クリックで月移動（表示データは3月固定でOK、UIの動きだけ実装）
- **部門フィルタ**（セレクトボックス）：「全部門」+部門リスト
- **表示切替**：「月表示」/「週表示」のトグルボタン（月表示のみ実装、週表示はトグルUIだけ存在すればよい）
- **シフト作成ボタン**（右端、プライマリーブルー）

#### 4.3.2 シフトカレンダー（メインエリア）

**横軸（カラム）：** 3月の日付（1日〜31日）。土曜は青文字、日曜・祝日は赤文字。  
**縦軸（行）：** 職員名（15名）。左端に固定カラムとして氏名を表示。

**セルの表記（勤務区分）：**

| 区分 | 表記 | 背景色 | テキスト色 |
|------|------|--------|-----------|
| 日勤 | 日 | `bg-blue-100` | `text-blue-700` |
| 夜勤 | 夜 | `bg-purple-100` | `text-purple-700` |
| 準夜勤 | 準 | `bg-indigo-100` | `text-indigo-700` |
| 休日 | 休 | `bg-gray-100` | `text-gray-400` |
| 有休 | 有 | `bg-amber-100` | `text-amber-700` |
| 当直 | 当 | `bg-red-100` | `text-red-700` |

**インタラクション：**
- セルをクリックすると、勤務区分を切り替えるポップオーバーを表示（日勤/夜勤/準夜勤/休日/有休/当直を選択可能）
- 選択するとセルの表記・色が即座に変わる
- 各列の下部に「日勤: X名 / 夜勤: X名」のサマリー行を表示

**ダミーシフトデータ：**
- 15名分 × 31日のシフトデータを生成する
- 医師は日勤+当直のパターン、看護師は日勤/夜勤/準夜勤のローテーション、事務職は日勤+土日休のパターンで生成
- 職種ごとの勤務パターンがリアリティのあるものになるようにすること

#### 4.3.3 シフトサマリーパネル（カレンダー右側または下部）

カレンダーの横、もしくは月表示下部に小さなカード群で表示。

- 今月の総勤務日数配分（日勤○○件 / 夜勤○○件 / ...）
- 部門別の未充足日（例：「救急科 3/12(水) 夜勤 1名不足」を赤テキストで）
- シフト確定ステータス（部門ごとに 確定済/未確定 のバッジ）

## 6. トランジション・アニメーション

デモアプリの品質感を高めるため、以下のアニメーションを実装する。

| 対象 | アニメーション | 仕様 |
|------|-------------|------|
| ページ遷移 | フェードイン | opacity 0→1、0.2s ease-in |
| モーダル表示 | フェード+スケール | opacity + scale 0.95→1、0.2s |
| スライドパネル | 右からスライド | translateX(100%)→0、0.3s ease-out |
| トースト通知 | 右上からスライドイン | translateX(100%)→0、0.3s、3秒後に自動消去 |
| カードホバー | シャドウ変化 | shadow-sm → shadow-md、0.2s |
| テーブル行ホバー | 背景色変化 | transparent → bg-blue-50、0.15s |
| ステータス変更 | バッジ色変化 | 0.3sでスムーズに色遷移 |
| サイドバーメニュー | アクティブ表示 | 左ボーダー+背景色を0.2sで遷移 |
| 数値カウントアップ | ダッシュボードKPI | ページ表示時に0から目標値まで0.8sでカウントアップ |

### 7.2 状態管理
- React の `useState` / `useContext` のみで管理。外部状態管理ライブラリは不要。
- シフトの編集状態、フィルタ条件、モーダルの開閉状態などをローカルステートで保持。

### 7.3 パフォーマンス
- ダミーデータは全て `data/mockData.ts` にまとめ、各ページで import して使用。
- 画像ファイルは一切使用しない。アバターはイニシャル表示で対応。

- Vercel にデプロイし、URLを取得する。
- `vercel.json` は不要（Next.js のデフォルト設定で動作）。
- 環境変数は不要。

## 8. 成功基準

このデモアプリが以下を達成できていれば成功とする。

1. **URLを開いた瞬間に「おっ」と思わせる**: ダッシュボードのKPIカード + チャートが視覚的に印象的であること
2. **触りたくなるUI**: クリック、フィルタ、シフト編集などの操作が気持ちよく動くこと
3. **病院業務への理解が伝わる**: 夜勤・当直・準夜勤といった医療特有の概念がUIに反映されていること
4. **本実装を想像できる**: 「このまま作り込めば本番で使える」とクライアントが感じられる構成であること
5. **3分で全容を把握できる**: オンライン面談で画面共有しながら3分でウォークスルーできるボリュームであること
